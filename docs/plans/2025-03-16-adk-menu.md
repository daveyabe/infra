# ADK Bash Menu Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** One bash script with subcommands and an interactive menu to init venv, run ADK CLI/web, resume from session JSON, create agent, and manage agent path (env/config).

**Architecture:** Single executable `scripts/adk-menu.sh`. Config: `ADK_AGENT_PATH` env, then `./.adk-agent-path`, then `~/.adk-menu.conf`. Venv at `$ADK_AGENT_PATH/.venv`. Subcommands: init, run, web, resume, create, path, help; no args = interactive menu.

**Tech Stack:** Bash (assume 4+ for arrays if used; otherwise POSIX-style), `adk` CLI (google-adk), standard Unix tools.

**Design reference:** `docs/plans/2025-03-16-adk-menu-design.md`

## Task 1: Script skeleton and config loading

**Files:**
- Create: `scripts/adk-menu.sh`

**Step 1: Create script with shebang and usage**

- Shebang: `#!/usr/bin/env bash`
- No `set -e` at top (menu loop must not exit on one failure).
- Usage function that prints:
  - Script name and one-line description.
  - "Usage: adk-menu.sh [command]"
  - Commands: init, run, web, resume, create, path, help.
  - "With no command, runs interactive menu."
  - "Config: ADK_AGENT_PATH env, or ./.adk-agent-path, or ~/.adk-menu.conf (line: ADK_AGENT_PATH=/path)."

**Step 2: Add load_config function**

- If `ADK_AGENT_PATH` already set and non-empty, return.
- Else if `./.adk-agent-path` exists and is readable, source or read it (parse only `ADK_AGENT_PATH=...`, ignore comments and blank lines), set `ADK_AGENT_PATH`.
- Else if `~/.adk-menu.conf` exists, same parsing, set `ADK_AGENT_PATH`.
- Export `ADK_AGENT_PATH` so child processes see it.

**Step 3: Add subcommand dispatch**

- Parse first argument as subcommand: init|run|web|resume|create|path|help.
- If no argument, set subcommand to `menu`.
- Case statement: call corresponding function (stubs that print "Not implemented: <cmd>" for now). For `help`, call usage and exit 0.

**Step 4: Verify**

- Run: `./scripts/adk-menu.sh help` → usage printed.
- Run: `./scripts/adk-menu.sh init` → "Not implemented: init".
- Run: `./scripts/adk-menu.sh` → "Not implemented: menu" (or menu stub that exits).

**Step 5: Commit**

## Task 2: Implement path and ensure_path helpers

**Files:**
- Modify: `scripts/adk-menu.sh`

**Step 1: ensure_path function**

- Call `load_config`.
- If `ADK_AGENT_PATH` is empty: print "ADK_AGENT_PATH not set. Use 'path' or set ADK_AGENT_PATH."; return 1.
- Return 0 otherwise. Used by init, run, web, resume.

**Step 2: path subcommand**

- Call `load_config`.
- Print "ADK_AGENT_PATH=${ADK_AGENT_PATH:-<not set>}".
- If interactive (e.g. when called from menu), prompt "Set path? [y/N]" and if yes, read path, export ADK_AGENT_PATH, then prompt "Save to config? [y/N]" and if yes, write `ADK_AGENT_PATH=<value>` to `~/.adk-menu.conf` (or create file). When run as subcommand (non-interactive), only show path and exit.

For simplicity, path subcommand can always prompt when path is not set; when path is set, just show it unless we add a flag like `--set`. Keep behavior: path = show; if not set, prompt to set and optionally save.

**Step 3: Wire path and help**

- Replace path stub with above. help already shows usage.

**Step 4: Verify**

- Call `ensure_path`; on failure exit 1.
- If `$ADK_AGENT_PATH` is not a directory, print "Agent path does not exist: $ADK_AGENT_PATH" and exit 1.

## Task 5: Implement web subcommand

**Files:**
- Modify: `scripts/adk-menu.sh`

**Step 1: web_subcommand function**

- Same ensure_path and path/venv checks as run.
- Determine parent of agent: `AGENT_PARENT=$(dirname "$ADK_AGENT_PATH")`, `AGENT_NAME=$(basename "$ADK_AGENT_PATH")`.
- Source venv activate, then `cd "$AGENT_PARENT"` and run `adk web --port 8000`. ADK docs: run adk web from parent directory that contains the agent folder. So command is `adk web --port 8000` from parent dir; ADK discovers agents in current dir. Confirm: "run from the parent directory that contains your my_agent/ folder". So we cd to parent and run adk web; agent dir name is the folder name. No need to pass path if adk web discovers by cwd. Run: `(cd "$AGENT_PARENT" && adk web --port 8000)` with venv activated in that subshell, or source venv then cd and adk web (same process). Prefer: source activate, cd "$AGENT_PARENT", exec adk web --port 8000 (so one process, correct cwd).

**Step 2: Wire web**

- Replace web stub with web_subcommand.

**Step 3: Verify**

- With real agent, run `./scripts/adk-menu.sh web` → server starts; open localhost:8000 and select agent.

**Step 4: Commit**

## Task 8: Implement interactive menu loop

**Files:**
- Modify: `scripts/adk-menu.sh`

**Step 1: menu_loop function**

- Loop: load_config; show "ADK Menu (agent: ${ADK_AGENT_PATH:-<not set>})"; print options 1–7 as in design; read choice.
- 1 → init (call init_subcommand; if ensure_path fails, prompt for path first via path flow then retry or ask to use option 6).
- 2 → run; 3 → web; 4 → resume. For 2–4, if ensure_path fails, print "Set agent path first (option 6)." and continue loop.
- 5 → create; 6 → path (call path subcommand in interactive way); 7 → exit 0.
- Invalid choice → print "Invalid choice" and continue.
- After each action (except Exit), press Enter to continue then loop again. Don't use set -e so a failed subcommand doesn't exit the script.

**Step 2: Wire no-args to menu_loop**

- When subcommand is `menu`, call menu_loop.

**Step 3: Verify**

- Run `./scripts/adk-menu.sh` → menu appears; choose 7 → exits. Choose 6 → path flow. Choose 1 with path set → init runs.

**Step 4: Commit**

```bash
git add scripts/adk-menu.sh
git commit -m "feat(adk-menu): implement interactive menu loop"
```
